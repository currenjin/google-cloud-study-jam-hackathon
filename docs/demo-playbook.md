## 목적

해커톤 당일에 구현이 조금 덜 되거나 API/배포가 실패해도, **문제 정의와 데모 흐름이 끊기지 않게** 준비한다.

## 1. 데모 시나리오

### 상황

Study Jam 참가자가 Cloud Run과 Gemini API 실습을 끝냈다. 하지만 실습 후 머릿속에 남은 내용이 흩어져 있다.

### 사용자 입력

```text
오늘 Cloud Run으로 컨테이너 앱을 배포하는 실습을 했다.
서비스 계정, region, allow unauthenticated 옵션이 헷갈렸다.
Gemini API는 Python에서 호출해봤지만 에러 처리와 환경변수 관리가 아직 익숙하지 않다.
해커톤에서는 배포가 실패해도 로컬 데모를 보여줄 수 있게 준비하고 싶다.
```

### 학습 목표

```text
해커톤에서 Study Jam Buddy를 로컬로 안정적으로 실행하고, 가능하면 Cloud Run으로 배포하기
```

### 자신감

```text
보통
```

## 2. 기대 출력 예시

### 5줄 요약

1. Cloud Run은 컨테이너 앱을 서버 관리 없이 배포하는 서비스다.
2. 배포 시 region, 인증 공개 여부, 서비스 계정 설정이 중요하다.
3. Gemini API는 학습 메모를 구조화된 복습 자료로 바꾸는 핵심 기능이다.
4. API 키와 환경변수는 Git에 커밋하지 않고 안전하게 관리해야 한다.
5. 해커톤에서는 Cloud 배포 실패를 대비해 로컬 fallback 데모가 필요하다.

### 복습 퀴즈

1. Cloud Run에서 `--allow-unauthenticated` 옵션은 무엇을 의미하는가?
2. region을 `asia-northeast3`로 선택하는 이유는 무엇인가?
3. API 키를 코드에 직접 넣지 않고 환경변수로 관리하는 이유는?
4. Gemini API가 실패했을 때 fallback이 필요한 이유는?
5. Study Jam Buddy에서 Firestore를 당장 쓰지 않는 이유는?

### 약점 개념

- IAM / 서비스 계정
- Cloud Run region / 배포 옵션
- 환경변수와 API 키 관리

### 다음 실습 체크리스트

- [ ] API 키 없이 fallback 모드로 앱 실행
- [ ] 샘플 학습 메모 2개 준비
- [ ] Gemini API 실제 호출은 1회만 테스트
- [ ] Cloud Run 배포 명령을 읽고 의미 설명
- [ ] 배포 실패 시 로컬 데모 전환 멘트 연습

### 오늘 30분 액션

> `streamlit run src/app.py`로 로컬 앱을 실행하고, 위 샘플 입력을 넣어 fallback 결과가 발표 가능한지 확인한다.

## 3. 3분 발표 흐름

### 0:30-1:00 해결책

> Study Jam Buddy는 학습 메모를 입력하면 Gemini가 5줄 요약, 복습 퀴즈, 약점 개념, 다음 실습 체크리스트, 오늘 30분 액션을 생성하는 AI 학습 코치입니다.

### 1:00-2:10 데모

1. 학습 메모 입력
2. 학습 목표 입력
3. 자신감 선택
4. 결과 생성 버튼 클릭
5. 요약/퀴즈/약점/체크리스트/30분 액션 확인

### 2:40-3:00 마무리

> 핵심은 AI가 답만 주는 것이 아니라, 배운 내용을 다음 행동으로 바꾸게 하는 것입니다. Study Jam Buddy는 학습과 해커톤 실천 사이의 간극을 줄입니다.

## 4. 장애 상황별 fallback

### Gemini API 실패

증상:

- API key 없음
- quota/rate limit
- 네트워크 에러

대응 멘트:

> 현재 데모는 API 장애를 대비해 fallback 모드가 준비되어 있습니다. 실제 운영에서는 Gemini API를 사용하지만, 해커톤 발표 안정성을 위해 같은 출력 schema의 로컬 결과를 보여드리겠습니다.

### Cloud Run 배포 실패

증상:

- API enable 지연
- 권한 문제
- 빌드 실패
- region 설정 문제

대응 멘트:

> Cloud Run 배포는 공식 크레딧과 권한이 확인되면 진행 가능한 확장 단계입니다. 지금은 로컬 Streamlit 앱으로 핵심 사용자 경험을 먼저 보여드리고, 컨테이너 기반 구조라 Cloud Run으로 이전 가능합니다.

### 인터넷 불안정

대응:

- 로컬 앱 실행
- fallback 결과 사용
- 샘플 입력/출력 캡처 준비

멘트:

> 네트워크와 외부 API 상태와 무관하게 핵심 데모가 가능하도록 fallback을 포함했습니다.

## 6. 심사 질문 대비

### Q. 왜 Study Jam Buddy가 필요한가?

A. Study Jam 참가자는 실습은 많이 하지만 학습 후 정리와 다음 행동이 약해지기 쉽다. 이 앱은 학습 메모를 복습 퀴즈와 다음 실습 액션으로 바꿔 학습 지속성을 높인다.

### Q. Google Cloud를 어디에 썼나?

A. 핵심 AI 변환은 Gemini API를 사용한다. 앱은 Cloud Run 배포 가능한 구조이며, 확장 시 Firestore에 학습 기록을 저장하고 Cloud Storage에 학습 자료를 저장할 수 있다.

### Q. 왜 Firestore/Storage를 이번 MVP에 안 넣었나?

A. 120분 해커톤에서는 저장 기능보다 입력 → AI 처리 → 학습 액션 출력이 핵심이다. 저장소는 확장 포인트로 두고, 데모 안정성과 완성도를 우선했다.

### Q. API나 배포가 실패하면?

A. fallback 모드가 있어 같은 출력 schema로 로컬 데모가 가능하다. 해커톤에서는 외부 의존성 실패에 대비하는 것도 중요한 완성도라고 봤다.

### Q. 실제 사용자에게 어떻게 확장할 수 있나?

A. Google Skills 학습 기록, Google Sheets, Calendar, Firestore를 연결해 개인별 복습 주기와 다음 실습 계획을 자동 관리하는 학습 코치로 확장할 수 있다.
