## 1. Project Description 초안

**릴드 (Reels Drama)** — 한 줄만 쓰면 다음 화가 나오는 릴레이 AI 드라마

GenZ는 하루 몇 시간씩 숏폼을 보지만, 만드는 사람은 극소수입니다. 촬영·편집·배우라는 창작 장벽 때문입니다. 릴드는 그 장벽을 문장 하나로 낮춥니다. 시드 한 줄로 드라마가 시작되고, 다음 사람이 전개 한 줄("갑자기 기억상실에 걸린다")을 얹으면 AI가 다음 화를 만듭니다. 기기를 손에서 손으로 넘기는 릴레이로, 소비자가 공동 창작자가 됩니다.

기술적으로는 두 가지가 핵심입니다. 첫째, **Story Bible** — 첫 시드에서 Gemini가 인물·배경·톤을 JSON으로 고정하고 모든 생성 호출에 주입해, 생성형 AI의 고질병인 캐릭터 일관성 문제를 해결했습니다. 인물은 얼굴이 아니라 소품(빨간 목도리, 파란 앞치마)으로 식별됩니다. 둘째, **목각인형 디오라마 아트 디렉션** — 무표정 인형이 막장 대사를 치는 갭 자체가 밈이 되고, 캐릭터 드리프트를 디자인으로 흡수합니다.

K-드라마의 나라 서울에서, 모두가 만드는 드라마. 이것이 릴드입니다.

- 트랙: Gemini in Entertainment
- 사용 기술: Gemini (각본·Story Bible 생성, structured output), Gemini 이미지 생성 (9:16 컷), Google AI Studio (API), Antigravity (개발)

```markdown
# 릴드 (Reels Drama) 🎬

> 한 줄만 쓰면 다음 화가 나오는 릴레이 AI 드라마

[데모 영상 보기](여기에_링크)

## What is this?

(위 Project Description 요약 2~3문장)

## How it works

시드 한 줄 → Gemini가 Story Bible(인물·배경·톤 JSON) 생성
→ 참가자가 전개 한 줄 입력 → Gemini가 바이블 기반 다음 화 각본
→ Gemini 이미지 생성이 목각인형 디오라마 컷 렌더 → 완성 릴 재생

캐릭터 일관성: Story Bible을 모든 호출에 주입 + 인물을 marker(소품/색)로 식별

## Tech

- Vite + React
- Gemini (모델ID) — 각본/바이블, structured output + 런타임 검증
- Gemini (이미지모델ID) — 9:16 컷 생성
- 실패 폴백: 이미지 실패 시 내레이션 텍스트 카드 (데모가 죽지 않음)

## Run

npm install
cp .env.example .env  # AI Studio API 키 입력
npm run dev

## Roadmap

공유 링크로 비동기 릴레이 · Veo 시네마틱 모드 · 팀 시즌제

## Screenshots

(시드 화면 / 턴 화면 / 릴 재생 3장)
```
