# 수상을 위해 준비해야 할 것

심사 기준 3가지에 하나씩 대응한다. 총상금 $4,000 — 1등 $2,000 크레딧 + Fitbit Air, People's Choice도 있음(= 다른 참가자에게 어필하는 것도 전략).

## 1. Technical Demo — "에러 없이 도는가"

이 기준이 1번인 이유: **120분 해커톤에서는 대부분의 팀 데모가 어딘가에서 죽는다.** 안 죽는 것만으로 상위권.

- 데모 영상은 **잘 도는 시점에 미리 녹화** (90분 시점 1차 녹화, 시간 남으면 재촬영)
- 라이브 시연 요청에 대비해 **행복 경로(happy path) 하나를 3번 반복 테스트**
- Veo 등 느린/불안정한 기능은 데모 경로에서 제외하고 "미리 생성한 결과"로 보여준다
- 실패 시 텍스트 카드 폴백 — 죽는 대신 연출처럼 보이게

## 2. Impact — "실제 문제를 해결하는가"

엔터테인먼트 트랙에서 Impact는 "문제 해결"보다 **"새로운 경험의 크기"**로 프레이밍한다.

발표 논리:
1. **소비만 하는 숏폼** — GenZ는 하루 몇 시간씩 릴스를 보지만, 만드는 사람은 극소수. 창작 장벽(촬영, 편집, 배우)이 높아서
2. **릴드는 소비자를 공동 창작자로** — 한 줄만 쓰면 드라마의 다음 화가 나온다. 창작 장벽을 문장 하나로 낮춤
3. **한국 콘텐츠 파워와 연결** — K-드라마 문법 × AI 생성 × 참여형 = 서울에서 시작할 이유가 있는 서비스 (트랙 설명의 "Korea—particularly Seoul—is a global powerhouse in entertainment" 그대로 회수)
4. **바이럴 루프가 내장됨** — 완성된 릴을 공유하면 보는 사람이 다음 턴에 참여하고 싶어짐 → 셀로그처럼 팀 단위 놀이문화 가능

## 3. Creativity — "새로운 접근인가"

"AI로 영상 만들기"는 흔하다. 차별점을 명확히 말로 박는다:

- **릴레이 = 사람이 플롯의 주도권을 가짐.** AI가 다 만드는 게 아니라, 사람들의 한 줄 전개가 스토리를 조종한다 (AI는 연출가, 사람이 작가)
- **Story Bible로 일관성 문제를 정면 해결** — 심사위원이 아는 생성 AI의 고질병을 해결했다고 명시적으로 언급
- **목각인형 디오라마는 한계가 아니라 선택** — 캐릭터 드리프트를 디자인으로 흡수 + 무표정 인형이 막장 대사를 치는 갭 자체가 밈. "저품질"이 아니라 "아트 디렉션"으로 발표
