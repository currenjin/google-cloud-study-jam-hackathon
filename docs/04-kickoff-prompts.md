# 킥오프 프롬프트 (Antigravity에 순서대로 붙여넣기)

당일 순서: Antigravity 로그인(**프로젝트 ID 지정 필수 — Business Plan**) → 이 레포 폴더 열기 → `.env` 파일에 API 키 넣기 → 아래 프롬프트를 순서대로 붙여넣는다.

```bash
# 레포 클론 후 (터미널에서 딱 이것만)
cp .env.example .env
# .env 열어서 AI Studio에서 받은 키 입력
```

---

## M1 프롬프트 (붙여넣기 → 약 20분)

```
AGENTS.md와 docs/03-prompts.md를 읽고 시작해.

M1을 구현해줘: Vite + React 프로젝트를 이 폴더에 스캐폴드하고,
다음 한 사이클이 동작하게 만들어.

먼저 demo-assets/mock-scenes.json의 목업 데이터로 화면 렌더링부터 완성한 뒤,
실제 Gemini 호출로 교체해 (UI 확인을 API 대기 없이 하기 위해).

1. 시드 입력 화면: 텍스트 입력 + "드라마 시작" 버튼
2. 버튼 클릭 → docs/03-prompts.md의 "프롬프트 1"로 Story Bible(JSON) 생성
3. 바이블 생성 직후 "프롬프트 2"로 첫 씬 생성 (새 전개 입력은 "이야기를 시작한다"로 고정)
4. 씬의 cuts[0].image_prompt로 이미지 1장 생성 (9:16)
5. 결과 화면: 이미지 + 씬 제목 + 내레이션 + 대사 표시

Gemini 호출 모듈은 src/gemini.js 하나로 모아줘 (바이블 생성 / 씬 생성 / 이미지 생성 함수 3개).
완성되면 npm run dev로 띄워서 시드 "재벌집 막내인형과 편의점 알바인형의 계약연애"로
동작 확인하고 커밋해.
```

## M2 프롬프트 (M1 확인 후 붙여넣기 → 약 40분)

```
M2를 구현해줘: 릴레이 턴 기능.

1. 씬 결과 화면 아래에 "다음 전개 한 줄" 입력창 + "다음 화 만들기" 버튼
2. 제출하면 프롬프트 2로 다음 씬 생성 — 이때 바이블 + 최신 story_summary + 입력한 전개를 전달
3. 씬이 생성될 때마다 updated_story_summary로 상태를 갱신하고 씬 배열에 누적
4. 씬 사이에 "📱 다음 사람에게 기기를 넘겨주세요" 오버레이 화면 (탭하면 입력창 노출) — 릴레이 느낌의 핵심
5. 전체 상태(바이블, 씬 배열, summary)를 localStorage에 백업하고 새로고침 시 복구

씬 3개까지 이어지는 걸 확인하고 커밋해.
```

## M3 프롬프트 (M2 확인 후 붙여넣기 → 약 30분)

```
M3를 구현해줘: 완성 릴 재생 + 폴백 + 폴리시.

1. "릴 재생" 버튼 → 9:16 세로 프레임 전체화면 재생 화면
   - 컷 이미지가 컷당 3.5초 자동 넘김, 하단에 내레이션/대사 자막 오버레이
   - 씬 전환 시 씬 제목(N화)을 잠깐 표시, 마지막에 "다음 화를 이어가세요" 엔딩 카드
2. 이미지 생성 실패 폴백: 1회 재시도 후 실패하면 어두운 배경 + 내레이션 타이포의
   텍스트 카드로 그 컷을 대체 (에러 노출 금지)
3. 모든 생성 대기에 로딩 상태 ("촬영 중...", "대본 쓰는 중..." 콘셉트 문구)
4. 전체 UI를 다듬어줘: 다크 배경, 세로 프레임 중심 레이아웃, 드라마 포스터 느낌의 타이틀

시드부터 릴 재생까지 전체 플로우를 한 번 돌려 확인하고 커밋해.
```
